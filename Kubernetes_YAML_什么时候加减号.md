# Kubernetes YAML：什么时候加 `-`

## 一句话规则

-   **`-`：表示数组（List）的一个元素**
-   **不加 `-`：表示对象（Map）的属性**

------------------------------------------------------------------------

## 判断方法

> **这个字段能不能有多个？**

-   能有多个 → **加 `-`**
-   只有一个 → **不加 `-`**

------------------------------------------------------------------------

## 常见示例

### 1. env（多个环境变量）✅ 加 `-`

``` yaml
env:
- name: DB_HOST
  value: mysql

- name: DB_PORT
  value: "3306"
```

因为可以配置多个环境变量。

------------------------------------------------------------------------

### 2. volumes（多个卷）✅ 加 `-`

``` yaml
volumes:
- name: data
  hostPath:
    path: /data
    type: Directory

- name: config
  configMap:
    name: app-config
```

因为一个 Pod 可以有多个 Volume。

------------------------------------------------------------------------

### 3. hostPath（一个对象）❌ 不加 `-`

``` yaml
hostPath:
  path: /data
  type: Directory
```

`hostPath` 只有一个对象，里面只有 `path`、`type` 等属性。

------------------------------------------------------------------------

## 常见需要 `-` 的字段

-   containers
-   initContainers
-   env
-   ports
-   volumeMounts
-   volumes
-   imagePullSecrets
-   hostAliases

------------------------------------------------------------------------

## 常见不需要 `-` 的字段

-   metadata
-   spec
-   hostPath
-   resources
-   securityContext
-   selector
-   labels

------------------------------------------------------------------------

## 记忆口诀

> **多个东西排队（数组）→ 加 `-`**\
> **一个东西带属性（对象）→ 不加 `-`**
